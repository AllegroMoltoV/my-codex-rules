## 実行手順

まず、AGENTS.md 記載のルールを把握してください。
そのうえで、上記の要件を読み取り、以下の手順を順番に実行してください。

### 1. ディレクトリの作成

プロジェクトルートに以下のディレクトリを作成してください。

- `.appendix` - 参考資料やブログ下書きなど、リポジトリに含めない補足ファイルの格納先
- `.tmp` - コマンド出力のリダイレクト先など、一時ファイルの格納先
- `.logs` - ログファイルの一次保管先
- `docs/reports` - 調査レポートの蓄積先
- `docs/logs` - 実行ログの蓄積先

### 2. 現状の調査

要件に関連する情報を、非破壊的な操作で可能な限り収集してください。

- ファイルの読み取り、ディレクトリ構造の確認、`git log` の参照など
- MCP サーバーが利用可能な場合は GET 系の操作で外部情報も収集する
- API ドキュメント、既存の設定ファイル、依存関係の確認なども含む

調査結果は `docs/reports/` に保存してください。

### 3. 要件の不明点の洗い出し

ステップ 2 の調査を踏まえてもなお不足している情報や曖昧な点を洗い出してください。
調査で判明済みの事項は質問に含めないでください。

洗い出した内容は `.prompts/QUESTIONS.md` に質問形式で保存してください。
