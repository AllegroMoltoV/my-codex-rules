# my-codex-rules

Codex で使う `AGENTS.md` のルール集です。

## 背景

Claude Code では `.claude/rules/` にルールを置いていましたが、Codex では `AGENTS.md` を使います。そのため、このリポジトリでは対象プロジェクトへコピーするための `AGENTS.md` を管理します。

## 内容物

### rules/AGENTS.md

対象プロジェクトのルートにコピーして使用するファイルです。

### .prompts/

対象プロジェクトの `.prompts/` にコピーして使用するファイルです。

| ファイル | 概要 |
|---|---|
| `INIT.md` | 初回セットアップ用 prompt。要件の記入欄、作業ディレクトリの作成、現状調査、質問の洗い出しを指示する |

### その他

| ファイル | 概要 |
|---|---|
| `exclude` | `.git/info/exclude` の参照用コピー |

## 使い方

### ルールの配置

`rules/AGENTS.md` を、対象プロジェクトのルートへ `AGENTS.md` としてコピーしてください。

### Git 除外設定

`git init` 実行後、`.git/info/exclude` の内容を本リポジトリの `exclude` で置き換えてください。`.prompts`、`AGENTS.md` などの Codex 関連ファイルが Git の追跡対象から除外されます。

### 初回セットアップ

対象プロジェクトに `.prompts/` ディレクトリを作成し、`.prompts/INIT.md` をコピーしてください。

要件セクションを記入したうえで、Codex に `.prompts/INIT.md` の実行を依頼してください。作業ディレクトリの作成と要件の不明点の洗い出しが行われます。
不明点の洗い出し前には非破壊的な現状調査を行い、調査結果を `docs/reports/` に保存します。

## 注意点

Windows では、Codex が作成した新規リポジトリで `git` の ownership 警告が出ることがあります。その場合は、対象ディレクトリを `safe.directory` に追加してください。
